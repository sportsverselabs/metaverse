# workflows/

Multi-step automations that coordinate multiple agents and/or integrations.
**Built in Phase 4** — nothing here yet.

A workflow describes: a trigger, the steps, which agents/integrations are involved,
and any owner-approval gates. Keep workflows portable and inspectable (defined as files
inside the project, not in an external proprietary system) unless the owner approves otherwise.

> Do not build workflows until agents and integrations exist and the owner advances to Phase 4.
